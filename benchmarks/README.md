# benchmarks/ — MACE-MH acceleration benchmarks

Measures and validates speed-ups for the MACE-MH foundation model on GPU:
`float64` → `float32`, fused **cuEquivariance** kernels, and `torch.compile`
(the "0.3.16 unblock" experiment). All optimisations are checked to be
**bit-identical** to the reference within rounding.

## Files
- `bench_mace.py` — core timing harness (energies / forces / stress, fp64 vs fp32).
- `gen_structures_large.py` — generate larger benchmark cells.
- `compare_runs.py` — diff two runs' energies/forces/stress within tolerance
  (`--etol/--ftol/--stol`).
- `mace_036_compile/` experiment (the `_036` tag = the MACE 0.3.16 unblock):
  `probe_compile_036.py`, `bench_compile_036.py`, `validate_036.py` (+ `_worker`),
  `build_master_table_036.py`.
- `validate_mptrj_cueq.py`, `validate_opt4.py` — numerical-equivalence proofs for
  the cuEquivariance path and the individual safe optimisations.
- `grid_036.sbatch` / `run_grid_036.sh` — grid sweep drivers (the `.sbatch` version
  is restart-safe; prefer it).
- `slurm/run_bench.sbatch`, `slurm/run_cueq_bench.sbatch` — SLURM launchers.

The `patches/mace_safe_opts.patch` (see `../patches/`) contains the bit-identical
edits these benchmarks validate.

## Required environment variables
`ZBL_MACE_MH1` (model), `ZBL_MPTRJ_XYZ` (validation structures), `ZBL_PYTHON` /
`ZBL_SLURM_PARTITION` for the launchers. See `config.example.env`. Benchmark
output JSON and logs are regenerated by these scripts and are gitignored.
